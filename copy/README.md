# Motor de Copy — kit do projeto

Essência extraída do **MSE Funnel Labs V2** (skill de copy/funil do Paulo), instalada aqui para escrever a copy desta proposta com método — e não no improviso.

## O que tem aqui

| Arquivo | Para que serve |
|---|---|
| `BRIEFING-COPY.md` | **Comece por aqui.** Ângulo mestre (a "malandragem"), Big Idea, mecanismo, inimigo e objeções. Rege todo o deck. |
| `AVATAR.md` | Quem é o comprador (dono da transportadora) e quem é o usuário (família). Dores, desejos, ponto de ruptura. |
| `COMUNICACAO.md` | Tom de voz: como escrever e o que nunca escrever. |
| `referencia/anti-ia-blacklist.yaml` | Lista de clichês, palavras de IA e padrões proibidos. **Filtro obrigatório.** |
| `referencia/copy-frameworks.yaml` | PAS, PASTOR, AIDA, níveis de consciência, oferta Hormozi, mecanismo Todd Brown. |
| `referencia/hook-headline-formulas.yaml` | 12 tipos de hook + 8 categorias de headline + scoring. |
| `referencia/objection-handler-playbook.yaml` | Objeções por categoria + como quebrar (LAER). |
| `referencia/copy-review.md` | Checklist de revisão antes de aprovar qualquer peça. |

## Regra do projeto
Nenhuma copy (deck, e-mail, página, anúncio) vai pra frente sem:
1. Estar alinhada ao `BRIEFING-COPY.md` (ângulo de negócio, não emocional).
2. Passar pelo filtro `referencia/anti-ia-blacklist.yaml`.
3. Bater os fundamentos do `referencia/copy-review.md`.

> O framework completo (90+ arquivos, agentes, workflows) está fora do projeto, no zip original. Aqui ficou só o que serve pra escrever copy desta venda.
